في 2026، لم يعد سؤال المطورين المحترفين "هل تستخدم الذكاء الاصطناعي في البرمجة؟"، بل "كم من مهامك اليومية فوَّضتها لوكيل برمجي؟". في هذا المشهد المتشبع، برز اسم يثير فضول المتخصصين: **Kilo Code**. ليس لأن صوته أعلى، بل لأن رقم استخدامه لا يُمكن تجاهله.

في العام الجاري، تجاوز Kilo Code عتبة **2.18 تريليون رمز** تمت معالجتها، ليتربع بذلك على المرتبة الثانية عالمياً بين وكلاء البرمجة من حيث الحجم، خلف OpenClaw العملاق وحده. هذا الرقم ليس مجرد إحصائية تسويقية؛ إنه مؤشر حقيقي على أن أداة مفتوحة المصدر، بلا كيان شركة ضخم خلفها، استطاعت أن تصبح جزءاً من سير عمل المطورين اليومي، وأكثر من مجرد تجربة عابرة.

## الانتشار غير المسبوق: 2.18 تريليون رمز و 1.5 مليون مستخدم

الأرقام تحكي قصة لا تحتاج إلى تفسير. اعتباراً من أوائل 2026، يتحدث المطورون عبر الشبكات عن تجاوز عدد مستخدمي Kilo Code عتبة **1.5 مليون "Kilo Coder"** حول العالم، مع معالجة شهرية ضخمة تُقدر بـ 6.1 تريليون رمز، ورقم إجمالي تراكمي يتجاوز **25 تريليون رمز**. في منصة OpenRouter، التي تُعتبر سوقاً موثوقة لنماذج LLM، تُوّج Kilo Code كـ #1 coding agent، متفوقاً على العديد من المنافسين التجاريين الكبار.

ما يجعل هذه الأرقام ملفتة حقاً هو سرعة النمو. Kilo Code لم يكن موجوداً قبل عام واحد. بدأ المشروع في مارس 2025 كفكرة طموحة داخل مجتمع المصادر المفتوحة، لينطلق بسرعة فائقة حتى تصدر قائمة Product Hunt اليوم الأول. هذا الصعود السريع يوحي بوجود حاجة ماسة للسوق لم تكن تلبيها الأدوات السابقة: وكيل برمجي مفتوح المصدر، مرن، غير مقيّد بنموذج اشتراك صارم.

## من أين أتى Kilo Code؟ جذور fork وتعاون مؤسس GitLab

Kilo Code لم يُخلق من العدم. هو نتاج مباشر للتطور الطبيعي لمشاريع المصادر المفتوحة، حيث يبني على أكتاف من سبقه. تقنياً، Kilo Code هو **fork مباشر من Roo Code**، ولكنه يضم أيضاً مجموعة واسعة من الميزات المستوردة والمحسّنة من Cline، إلى جانب إضافاته الخاصة. هذا التراكم التقني جعله يمتلك قاعدة صلبة منذ اليوم الأول.

الأمر الأكثر إثارة للاهتمام هو هوية المؤسسين. Kilo Code ليس مشروعاً جانبياً لمبرمج متحمس، بل وراءه اسم ثقيل: **سيد سيجبرانديج (Sid Sijbrandij)**، المؤسس والرئيس التنفيذي السابق لـ GitLab، وانضم إليه سكوت برايتنوثر (Scott Breitenother) كرئيس تنفيذي. هذا الظهور المبكر لشخصيات بهذا الوزن أعطى المشروع مصداقية فورية. في ديسمبر 2025، جمع المشروع تمويلاً أولياً بقيمة **8 ملايين دولار**، مما يجعله واحداً من أكبر المشاريع مفتوحة المصدر في مجال الـ AI coding التي تحصل على تمويل مؤسسي.

منذ انطلاقته، تم وضع المشروع تحت تنظيم Kilo-Org على GitHub، مع فلسفة واضحة: مجتمعي، قابل للتوسع، ونموذج-agnostic. في فبراير 2026، اتخذت Kilo خطوة إضافية نحو الشفافية المطلقة بنشر كود "Gateway" والبنية الخلفية السحابية (Cloud backend) بالكامل تحت تراخيص source-available، مما أتاح للمجتمع التدقيق في كيفية عمل الأداة خلف الكواليس.

## نموذج العمل والمرونة المالية: كسر نموذج الاشتراك الشهري

السبب الثاني الذي يُفسر انتشار Kilo Code الواسع هو فلسفته الاقتصادية. بدلاً من فرض اشتراك شهري ثابت (مثل $10 أو $20 شهرياً كما تفعل Cursor و GitHub Copilot)، تتبنى Kilo Code نموذج "Pay What You Use".

يمكن للمطورين الاختيار من بين عدة خيارات: **إحضار مفتاح API الخاص بهم** (Bring Your Own Key) من مزود مثل Anthropic أو OpenAI أو Google، أو استخدام رصيد مدفوع مسبقاً ضمن النظام، أو حتى الاستفادة من نموذج محلي مجاني بالكامل يعمل على الجهاز (عبر Ollama أو LM Studio). بالنسبة لمن لا يريد عناء الإعداد الأولي، تقدم Kilo Code أيضاً "Kilo Pass" بخطط تبدأ من $19 شهرياً (Starter) وصولاً إلى $49 شهرياً (Pro)، مع طبقة مجانية تمنح Credits بقيمة $20 للبدء. هذا التنوع في خيارات الدفع جعلها جذابة لكل من المطور الفرد والشركة الناشئة التي تحاول التحكم في تكاليف السحابة.

تدعم الأداة حالياً أكثر من 500 نموذج LLM مختلف، من GPT-5 و Claude 4 Sonnet/Opus إلى Gemini 2.5 Pro والنماذج المحلية الصغيرة. عدم تقييد المستخدم بنموذج واحد يمنح المطورين حرية الاختيار بناءً على الميزانية والجودة المطلوبة.

## المعمارية التقنية: كيف يُعيد Kilo Code تعريف الوكيل البرمجي؟

ما يميز Kilo Code عن مجرد "مساعد برمجي" تقليدي هو أنه صُمم ليكون **وكيلاً مستقلاً** (autonomous agent)، وليس مجرد أداة لإكمال السطور.

في قلب الأداة توجد دورة عمل تُعرف بـ "Plan-Act-Observe-Fix". هذا يعني أن Kilo Code لا ينتظر تعليمات فورية بعد كل سطر؛ بل يُعطى مهمة معقدة، فيخطط خطواتها، ينفذها، يراقب النتائج، ثم يُصلح الأخطاء التي تنشأ، بشكل مشابه لكيفية عمل مطور بشري مبتدئ تحت الإشراف.

**فهم السياق الكامل (Context Awareness):** باستخدام ما يُعرف بـ "Memory Bank" الدائم، يحتفظ Kilo Code بفهمه لهيكل المشروع عبر الجلسات، حتى دون الحاجة لإعادة تحليل الملفات كل مرة.

**التعديل متعدد الملفات (Multi-file Editing):** قدرة الوكيل على إجراء تغييرات تمتد عبر 10 أو 20 ملفاً في طلب واحد، مع ضمان عدم كسر التبعيات بينها، تُعتبر الميزة التي تجعل المطورين المحترفين يتحولون إليه. عبر أدوات مثل `read_file`, `search_files`, `ApplyPatchTool`، يمكن لـ Kilo Code إعادة هيكلة وحدات برمجية كاملة دون تدخل بشري.

**التنفيذ المتوازي (Parallel Execution):** مع بنيته الجديدة المبنية على "Portable Core"، يمكن لـ Kilo Code تشغيل قراءة الملفات، البحث، وأوامر الطرفية بشكل متزامن، مما يقلل من زمن الانتظار.

## الأداء في الممارسة اليومية: ما يقوله المستخدمون

البيانات الصادرة عن التجارب الميدانية تُظهر أن Kilo Code يُحدث فرقاً كمياً في الإنتاجية. تشير إحدى الدراسات إلى أن استخدامه يقلل متوسط وقت الترميز بنسبة 78%، ويزيد كفاءة التصحيح بنسبة 300%، ويختصر وقت تأهيل المطورين الجدد في المشاريع الكبيرة من أسابيع إلى يومين فقط. هذا الكفاءة تفسر سرعة انتشاره.

في الاستخدام اليومي، يُستخدم Kilo Code في مهام روتينية مثل: إدارة التبعيات (dependency management)، تحديث التوثيق، إصلاح أخطاء الـ typing، وتحديث ملفات الترجمة. الحالات الاستخدامية المتقدمة تشمل بناء واجهات برمجة تطبيقات كاملة (API) من الصفر بناءً على تعليمات طبيعية، أو تحويل محادثة Slack بأكملها إلى كود وطلبات سحب (Pull Requests).

أحد الأمثلة الطريفة على قدراته هو استخدام "Architect Mode" لبناء موقع إخباري ساخر بالكامل، حيث قام Kilo Code بالتخطيط للمشروع، إنشاء الهيكل، وتنفيذ الكود، في جلسة واحدة.

## الحدود التي لم تُكسر بعد

مع كل هذه القوة، لا تخلو الأداة من عيوب. أكبر عيب وثّقه المستخدمون هو ضعف أداء Kilo Code عند دمجه مع **النماذج المحلية (Local Models)**، حيث تُظهر هذه النماذج صعوبة في استدعاء الأدوات بشكل موثوق، أو الاحتفاظ بسياق المهمة، مما يؤدي أحياناً إلى حلقات لا نهائية أو تجاهل المهام.

كما أشار بعض المستخدمين إلى أن واجهة المستخدم لا تزال تعاني من غموض في التصنيفات، حيث قد يصعب على المستخدم تمييز أي وكيل فرعي يعمل حالياً ("general agent" مقابل وكيل مخصص). مشاكل الاستقرار الدوري (خاصة مع تكامل بعض النماذج مثل xAI Grok) وُثقت أيضاً في تقارير المستخدمين.

على صعيد الأمان، بما أن الأداة تنفذ أوامر طرفية وتصل إلى نظام الملفات، فقد صنفها باحثو أمان بمستوى خطر "متوسط" (Medium Risk) بسبب احتمال تعريض النظام للخطر إذا أُسيء استخدامها، أو إذا تم اختراق المفاتيح API المستخدمة.

## خارطة الطريق والمستقبل

Kilo Code يعمل على بناء نظام بيئي متكامل حول الوكيل. من أبرز مبادراتهم **سوق MCP (Model Context Protocol)** الذي يسمح للمطورين بمشاركة "مهارات" وأدوات جاهزة، تشبه إلى حد كبير إضافات VS Code ولكن لوكلاء AI.

توسعت الأداة مؤخراً من VS Code إلى **JetBrains IDEs** و CLI، مما يسمح للمطورين بالبقاء في بيئتهم المفضلة.

في السوق المزدحم بأسماء مثل Cursor (الذي يعيد تعريف IDE نفسه) و Cline (الأصل البسيط)، يحتل Kilo Code مساحة فريدة: هو البديل مفتوح المصدر الذي يجمع بين قوة Cline ومرونة Roo Code، مع خلفية إدارية محترفة وسياسة تسعير لا تستنزف الميزانية.

مع توقعات بوصول عدد المطورين إلى 2 مليون قبل نهاية 2026، والاستمرار في تحسين تجربة النماذج المحلية، يبدو أن "Kilo Coders" ليسوا مجرد موضة عابرة، بل جماعة متنامية من المهنيين الذين اختاروا التحكم الكامل في أدواتهم بدلاً من الخضوع لنماذج الاشتراك المغلقة.
